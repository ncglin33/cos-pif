# Blueprint: PIF 製作系統

## 專案概覽

這是一個專為化妝品產業設計的「產品資訊檔案 (Product Information File, PIF)」管理系統。它提供了一整套網頁工具，用於建立、管理和匯出法規要求的 PIF 文件。此系統建立在一個安全的多使用者環境中，具有基於角色的存取控制，讓管理員可以有效地管理使用者註冊與權限。

## 核心功能與設計原則

### 風格與設計
- **現代化與簡潔**：使用者介面遵循現代美學，具有乾淨的佈局、充足的留白和一致的色彩規範。
- **元件化**：介面由模組化的元件（如卡片、按鈕、表單輸入）構成，以確保一致性與可重用性。
- **響應式設計**：應用程式被設計為完全響應式，確保在桌面和行動裝置上都能提供無縫的體驗。
- **互動式回饋**：使用者的操作會伴隨著清晰的視覺回饋，例如懸停效果、載入狀態和狀態訊息。
- **無障礙設計 (A11Y)**：設計融合了無障礙最佳實踐，包括語意化的 HTML、正確的標籤和鍵盤導航支援。

### 已實作功能
- **使用者身份驗證系統 (Authentication)**:
    - **註冊 (`register.html`)**: 新使用者可以建立帳戶。其初始狀態為 `pending`，等待管理員審核。
    - **登入 (`login.html`)**: 已註冊且被核准的使用者可以使用「電子郵件/密碼」登入。系統會驗證其 `status` 是否為 `active` 來決定是否授予存取權限。
    - **忘記密碼 (`forgot-password.html`)**: 使用者可以透過輸入其註冊的電子郵件來接收密碼重設連結，以重新設定密碼。
- **管理員後台 (`admin.html`)**:
    - **權限保護**: 只有 `role` 為 `admin` 的使用者才能存取此頁面。
    - **使用者管理**: 管理員可以批准、拒絕、延長效期。
    - **通知信箱設定**: 管理員可以設定 email，以便在新用戶註冊時收到通知。
- **後端與自動化 (Cloud Functions)**:
    - **新用戶註冊通知**: 自動發送 email 通知管理員。
    - **用戶狀態變更通知**: 自動發送 email 通知用戶其狀態變更（批准或拒絕）。
- **PIF 資料管理**:
    - **儀表板 (`pif-i.html`)**: 集中式的儀表板，用於檢視和管理所有 PIF 專案。
    - **PIF 導覽頁 (`pifs.html`)**: 每個 PIF 專案的導覽中心，可連結至所有文件部分。
    - **GMP 聲明 (`pif05.html`)**: 用於上傳 GMP 或 ISO 22716 證書。
    - **使用方法 (`pif07.html`)**: 結構化表單，用於詳細記錄產品使用方法、部位、用量等，並支援 PDF 上傳。
    - **不良反應 (`pif08.html`)**: 經過重構的整合頁面，專用於聲明產品上市後的不良反應，並提供穩定的 PDF 上傳、下載與刪除功能。
    - **理化資料模組**
        - **批次匯入 (`admin_import.html`)**: 管理員工具，可透過 CSV/JSON 檔案批次上傳成分的理化資料。
        - **報告頁面 (`pif09-2.html`)**: 自動產生 PIF 中所有成分的理化特性報告。
    - **毒理資料模組**:
        - **批次匯入 (`admin_import_toxic.html`)**: 管理員工具，可批次上傳成分的毒理資料。
        - **報告頁面 (`pif10.html`)**: 自動產生 PIF 中所有成分的毒理學特性報告。
    - **防腐效能試驗報告 (`pif13.html`)**: 結構化表單，用於記錄微生物挑戰測試結果，並支援上傳 PDF 格式的完整報告。
- **資料庫與安全性**:
    - **Firestore 資料庫**: 所有使用者和 PIF 資料都安全地儲存在 Firebase Firestore 中。
    - **精細的存取規則 (`firestore.rules`)**: 實作了嚴格的安全規則，確保只有授權使用者才能存取特定資料。

## 最新執行計畫：為 PIF 13 模組新增 PDF 上傳功能

### 目標
為「13 防腐效能試驗報告」頁面 (`pif13.html`) 新增一項功能，允許使用者上傳該頁面的 PDF 版本，並將其連結儲存於 Firestore，方便日後查閱與歸檔。

### 執行步驟
1.  **擴充 HTML 結構**:
    - **完成**: 在 `pif13.html` 的頂端工具列中，新增了「上傳本頁 PDF」按鈕及一個隱藏的檔案選擇器。
    - **完成**: 在頁面主體新增了一個區塊 (`#pdfUploadStatus`)，用於即時顯示上傳進度、成功狀態或錯誤訊息。

2.  **整合 JavaScript 上傳邏輯**:
    - **完成**: 匯入 Firebase Storage 的 `uploadBytesResumable` 函式以實現進度追蹤。
    - **完成**: 編寫了 `uploadPifPdf` 函式，負責處理檔案上傳、即時更新進度條，並處理可能發生的錯誤。
    - **完成**: 上傳成功後，自動獲取檔案的 `downloadURL`，並將其存入 Firestore `pifs/{pifId}` 文件中的 `pif13.pdfSummaryUrl` 欄位。
    - **完成**: 實現了上傳後自動觸發 `saveData`，確保資料同步。

3.  **更新設計藍圖**:
